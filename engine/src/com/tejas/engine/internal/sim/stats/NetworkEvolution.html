Network evolution

This report shows some metrics characterizing the evolution of the network during the simulation.

Important: Average values are computed in a time-averaged fashion, taking into account the period of time in each state.

Topology and link capacities

MetricValue
Number of nodes#topologyInfo_numberOfNodes#
Average number of links#topologyInfo_numberOfLinks#
Average node in/out degree#topologyInfo_nodeDegree#
Average network diameter (hops, km, ms)#topologyInfo_diameter#
Average total capacity installed (Erlangs, bps)#topologyInfo_totalCapacity#
  • Number of nodes: Number of nodes of the network. This value remains fixed during the simulation
  • Average number of links: Average number of links installed in the network during the simulation
  • Average node in/out degree: Average number of links per node
  • Average network diameter: Average length of the largest path among all-pairs shortest path. This metric is measured in hops, kilometers and miliseconds
  • Average total capacity installed: Average capacity installed in the whole network, summing the capacity in all the links

Traffic

MetricValue
Number of demands#trafficInfo_numberOfDemands#
Average total offered traffic (Erlangs, bps)#trafficInfo_totalTraffic#
Average offered traffic per node pair (Erlangs, bps)#trafficInfo_trafficPerNodePair#
Average blocked traffic (%)#trafficInfo_blockedTraffic#
Average over-subscribed traffic (%)#trafficInfo_oversubscribedTraffic#
  • Number of demands: Number of demands. This value remains fixed during the simulation
  • Average total offered traffic: Average offered traffic to the whole network, summing the offered traffic by every demand
  • Average offered traffic per node pair: Average offered traffic between each pair of nodes. It is equivalent to the ratio between the previous metric and the number of node pairs
  • Average blocked traffic: Average percentage of the offered traffic which is blocked
  • Average over-subscribed traffic: Average percentage of the carried traffic which is above the capacity of the links. In other words, it is the time-average of the summation of the excess traffic in each link divided by the total carried traffic

Routing

MetricValue
Average number of routes#routingInfo_numberOfRoutes#
Average bottleneck utilization (w. reserved bw, w.o. reserved bw)#routingInfo_bottleneckUtilization#
Average route length (hops, km, ms)#routingInfo_routeLength#
  • Average number of routes: Average number of established routes
  • Average bottleneck utilization (w. reserved bw, w.o. reserved bw): Average utilization of the bottleneck link in the whole network. This metric is computed considering or not the reserved bandwidth by protection segments (if any)
  • Average route length: Average length of the defined routes. This metric is measured in hops, kilometers and miliseconds

Protection segments

MetricValue
Average number of protection segments#segmentInfo_numberOfSegments#
Average link capacity reserved for protection (%, Erlangs, bps)#segmentInfo_capacityReserved#
Average carried traffic unprotected (%)#segmentInfo_unprotectedTraffic#
Average carried traffic with complete and dedicated protection (%)#segmentInfo_dedicatedProtectionTraffic#
Average carried traffic with partial and/or shared protection (%)#segmentInfo_sharedProtectionTraffic#
  • Average number of protection segments: Average number of defined protection segments
  • Average link capacity reserved for protection: Average capacity reserved for protection in each link. This metric is measured in percentage of the link capacity (summation of the total reserved bandwidth divided by the total link capacity), Erlangs and bps
  • Average carried traffic unprotected: Average percentage of traffic which is not protected by any segment
  • Average carried traffic with complete and dedicated protection: Average percentage of traffic which is totally protected by some dedicated protection segment(s), that is, which under a failure is able to carry the whole traffic carried by the primary path
  • Average carried traffic with partial and/or shared protection: Average percentage of traffic that does not fall into the two previous categories
